# v1.0 to v1.1.0 Node Migration Guide

**For Node Operators Upgrading from AMMOcoin v1.0**

**Date:** December 28, 2025
**Critical:** Read this ENTIRE guide before upgrading!

---

## ⚠️ **CRITICAL: Understanding Path A Migration**

AMMOcoin v1.1.0 uses **Path A migration strategy**, which means:

❗ **v1.1.0 is a COMPLETELY NEW BLOCKCHAIN**
❗ **v1.0 and v1.1.0 are SEPARATE, incompatible chains**
❗ **Your v1.0 wallet balances will NOT automatically appear in v1.1.0**
❗ **v1.0 private keys are REQUIRED to claim v1.1.0 balances**

### What This Means:

| Aspect | v1.0 | v1.1.0 |
|--------|------|--------|
| Genesis Block | `0x000001c4...` | `0x0000075a...` ✅ **Different** |
| Blockchain | Old chain | Fresh new chain |
| Your Wallet | Has balances | Starts empty |
| Migration | - | **Manual claim process** |

**DO NOT DELETE YOUR v1.0 WALLET!** You need it to prove ownership and claim v1.1.0 balances.

---

## 🔐 **Why v1.0 Wallets Are Critical**

Your v1.0 wallet contains the **private keys** that prove you owned AMMO on the v1.0 blockchain.

### You Need v1.0 Keys To:

1. **Prove Ownership** - Sign messages with your v1.0 addresses
2. **Claim v1.1.0 Balances** - Use the `verify-claim.py` script
3. **Receive Migration** - Top 61 holders get automatic distribution
4. **Access Migration Fund** - Small holders claim from migration fund

**Losing your v1.0 wallet = Losing access to v1.1.0 balances!**

## 📋 **Pre-Migration Checklist**

Before running the v1.1.0 setup script, complete these steps:

### ✅ **Step 1: Stop v1.0 Daemon**

```bash
ammocoin-cli stop

# Wait for clean shutdown
sleep 10

# Verify it's stopped
ps aux | grep ammocoind
```

### ✅ **Step 2: Backup v1.0 Wallet (CRITICAL!)**

```bash
# Full data directory backup
cp -r ~/.ammocoin ~/.ammocoin-v1.0-BACKUP-$(date +%Y%m%d)

# Extra wallet.dat backup
cp ~/.ammocoin/wallet.dat ~/wallet-v1.0-$(hostname)-$(date +%Y%m%d).dat
chmod 600 ~/wallet-v1.0-*.dat

# Verify backups exist
ls -lh ~/.ammocoin-v1.0-BACKUP-* ~/wallet-v1.0-*.dat
```

### ✅ **Step 3: Export Private Keys**

```bash
# If v1.0 daemon is running
ammocoin-cli dumpwallet ~/v1.0-wallet-dump-$(date +%Y%m%d).txt
chmod 600 ~/v1.0-wallet-dump-*.txt

# List all addresses with balances
ammocoin-cli listaddressgroupings > ~/v1.0-addresses-$(date +%Y%m%d).txt

# Get wallet info
ammocoin-cli getwalletinfo > ~/v1.0-wallet-info-$(date +%Y%m%d).txt
```

### ✅ **Step 4: Verify Backups**

```bash
# Check all backups exist
ls -lh ~/ | grep v1.0

# Expected files:
# .ammocoin-v1.0-BACKUP-YYYYMMDD/  (directory)
# wallet-v1.0-HOSTNAME-YYYYMMDD.dat
# v1.0-wallet-dump-YYYYMMDD.txt
# v1.0-addresses-YYYYMMDD.txt
# v1.0-wallet-info-YYYYMMDD.txt
```

### ✅ **Step 5: Secure Backups**

```bash
# Create encrypted archive (OPTIONAL but recommended)
tar -czf v1.0-complete-backup-$(date +%Y%m%d).tar.gz \
    ~/.ammocoin-v1.0-BACKUP-* \
    ~/wallet-v1.0-*.dat \
    ~/v1.0-wallet-dump-*.txt \
    ~/v1.0-addresses-*.txt \
    ~/v1.0-wallet-info-*.txt

# Secure the archive
chmod 600 v1.0-complete-backup-*.tar.gz

# CRITICAL: Copy this archive to a DIFFERENT machine!
# Use scp, external drive, or cloud backup
```

Once your v1.0 data is backed up:

```bash
cd ~
git clone https://github.com/everquin/AMMOcoin-v1.1.0.git
cd AMMOcoin-v1.1.0

# Run the setup script
sudo bash scripts/setup-seed-node.sh
```

### What the Script Does Automatically:

1. ✅ Detects existing v1.0 installation
2. ✅ Stops v1.0 daemon if running
3. ✅ Creates additional v1.0 backups

6. ✅ Installs to `/usr/local/bin/` (replaces old binaries)
7. ✅ Creates **SEPARATE** data directory: `~/.ammocoin-v1.1.0`
8. ✅ Leaves v1.0 data untouched at: `~/.ammocoin`
9. ✅ Sets up systemd service for auto-start

## 🔧 **Berkeley DB Compilation Issue**

### The Problem

Berkeley DB 4.8 has a compilation issue on modern Ubuntu with GCC 9+:

```
error: '__atomic_compare_exchange' was not declared in this scope
```

### The Solution (Automated in Script)

The setup script automatically applies this patch:

```patch
--- dbinc/atomic.h.orig
+++ dbinc/atomic.h
-	__atomic_compare_exchange((p), (o), (n))
+	__atomic_compare_exchange_db((p), (o), (n))
```

This renames Berkeley DB's atomic function to avoid conflict with GCC's built-in.

**This is standard, safe, and necessary for modern systems.**

## 📁 **Post-Installation Directory Structure**

After setup completes:

```
~/.ammocoin/              ← v1.0 data (PRESERVED)
  ├── wallet.dat          ← v1.0 wallet (KEEP THIS!)
  ├── blocks/             ← v1.0 blockchain
  ├── chainstate/         ← v1.0 state
  └── debug.log           ← v1.0 logs

~/.ammocoin-v1.1.0/       ← v1.1.0 data (NEW)
  ├── wallet.dat          ← v1.1.0 wallet (empty initially)
  ├── blocks/             ← v1.1.0 blockchain
  ├── chainstate/         ← v1.1.0 state
  ├── ammocoin.conf       ← v1.1.0 config
  └── debug.log           ← v1.1.0 logs

~/.ammocoin-v1.0-BACKUP-YYYYMMDD/  ← v1.0 backup
  └── (complete copy of v1.0 data)

~/wallet-v1.0-HOSTNAME-YYYYMMDD.dat  ← v1.0 wallet backup
~/v1.0-wallet-dump-YYYYMMDD.txt      ← v1.0 private keys
```

## 🔄 **Accessing v1.0 Data (If Needed)**

### Reading v1.0 Wallet (Without Running Daemon)

```bash
# The wallet.dat is still at:
ls -lh ~/.ammocoin/wallet.dat

# Or in backup:
ls -lh ~/.ammocoin-v1.0-BACKUP-*/wallet.dat
```

### Running v1.0 and v1.1.0 Simultaneously (Advanced)

If you need both running at once:

```bash
# v1.0 on custom ports (if you still have v1.0 binaries)
/path/to/old/ammocoind \
  -datadir=~/.ammocoin \
  -port=37021 \
  -rpcport=8333 \
  -daemon

# v1.1.0 on standard ports
sudo systemctl start ammocoind

# Access v1.0
/path/to/old/ammocoin-cli -datadir=~/.ammocoin -rpcport=8333 getinfo

# Access v1.1.0
ammocoin-cli -datadir=~/.ammocoin-v1.1.0 getinfo
```

## 💰 **Claiming v1.1.0 Balances**

### For Top 61 Holders (Automatic Distribution)

If you're in the top 61 holders:

1. Wait for the distribution transaction
2. Your v1.1.0 address will be **funded automatically**
3. Check balance:
   ```bash
   ammocoin-cli -datadir=~/.ammocoin-v1.1.0 getbalance
   ```

### For All Other Holders (Migration Fund Claim)

If you're not in top 61, use the claim process:

```bash
# 1. Get your v1.0 address that held AMMO
V1_ADDRESS="YOUR_V1.0_ADDRESS"

# 2. Get your v1.1.0 receiving address
V1_1_ADDRESS=$(ammocoin-cli -datadir=~/.ammocoin-v1.1.0 getnewaddress)

# 3. Sign a message with v1.0 private key
# (You'll need to import v1.0 private key or use old wallet)
MESSAGE="Claim AMMO migration from $V1_ADDRESS to $V1_1_ADDRESS"
SIGNATURE=$(ammocoin-cli -datadir=~/.ammocoin signmessage "$V1_ADDRESS" "$MESSAGE")

# 4. Submit claim
python3 scripts/verify-claim.py \
  --v1-address "$V1_ADDRESS" \
  --v1-1-address "$V1_1_ADDRESS" \
  --signature "$SIGNATURE"
```

**See `USER_MIGRATION_GUIDE_v1.0_to_v1.1.0.md` for detailed claim instructions.**

## 🔍 **Troubleshooting**

### Issue: "Berkeley DB won't compile"

**Solution:** The setup script includes the atomic.h patch. If it fails:

```bash
cd /tmp/db-4.8.30.NC
# Apply patch manually (see script for patch content)
patch -p0 < atomic.patch
cd build_unix
make clean
make -j$(nproc)
sudo make install
```

### Issue: "v1.1.0 node shows v1.0 blocks"

**Solution:** You're using the wrong data directory.

```bash
# Check which directory the daemon is using
ps aux | grep ammocoind

# Should show: -datadir=/home/USER/.ammocoin-v1.1.0

# If not, restart with correct directory:
sudo systemctl restart ammocoind
```

### Issue: "Can't find v1.0 wallet after upgrade"

**Solution:** v1.0 data is preserved.

```bash
# Check original location
ls -lh ~/.ammocoin/wallet.dat

# Check backup
ls -lh ~/.ammocoin-v1.0-BACKUP-*/wallet.dat

# Check individual backup
ls -lh ~/wallet-v1.0-*.dat
```

### Issue: "Genesis hash doesn't match"

**Solution:** Verify you're checking the v1.1.0 node:

```bash
# Get v1.1.0 genesis hash
ammocoin-cli -datadir=~/.ammocoin-v1.1.0 getblockhash 0

# Should be: 000005cb7068246016a7cc43aedde75eee3de551f24afca2b0dc28cfc4fb3329

# If different, you may be connected to wrong network
# Check peers:
ammocoin-cli -datadir=~/.ammocoin-v1.1.0 getpeerinfo
```

## 📊 **Verification Checklist**

After migration, verify everything is correct:

```bash
# 1. Check v1.0 backups exist
[ -d ~/.ammocoin-v1.0-BACKUP-* ] && echo "✓ v1.0 backup exists"
[ -f ~/wallet-v1.0-*.dat ] && echo "✓ Wallet backup exists"

# 2. Check v1.1.0 binaries
ammocoind --version | grep "v1.1.0" && echo "✓ v1.1.0 binary"

# 3. Check v1.1.0 data directory
[ -d ~/.ammocoin-v1.1.0 ] && echo "✓ v1.1.0 data dir exists"

# 4. Check v1.1.0 is running
systemctl is-active ammocoind && echo "✓ Daemon running"

# 5. Check v1.1.0 genesis
GENESIS=$(ammocoin-cli -datadir=~/.ammocoin-v1.1.0 getblockhash 0)
[ "$GENESIS" = "000005cb7068246016a7cc43aedde75eee3de551f24afca2b0dc28cfc4fb3329" ] && echo "✓ Correct genesis"

# 6. Check connections
PEERS=$(ammocoin-cli -datadir=~/.ammocoin-v1.1.0 getconnectioncount)
[ "$PEERS" -gt 0 ] && echo "✓ Connected to peers ($PEERS)"
```

All checks should show ✓

## 🆘 **Emergency Recovery**

If something goes wrong and you need to revert:

### Restore v1.0 from Backup

```bash
# Stop v1.1.0
sudo systemctl stop ammocoind

# Restore v1.0 data
rm -rf ~/.ammocoin
cp -r ~/.ammocoin-v1.0-BACKUP-YYYYMMDD ~/.ammocoin

# If you saved old v1.0 binaries, reinstall them
# Otherwise, rebuild v1.0 from source

# Start v1.0
ammocoind -daemon

# Verify
ammocoin-cli getinfo
```
